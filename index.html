Cargo Manifest // 02

Container vs. virtual machine

Trait Virtual Machine Docker Container
Boot time Minutes — boots a full OS Seconds — shares the host kernel
Size Gigabytes per VM Megabytes per image, typically
Isolation Full hardware-level isolation Process-level isolation
Portability Tied to a hypervisor Runs anywhere Docker runs

Cargo Manifest // 03

Your first Dockerfile

# Start from a small base image
FROM node:20-alpine

# Set the working directory inside the container
WORKDIR /app

# Copy dependency files first, install, then copy the rest
COPY package*.json ./
RUN npm install
COPY . .

# Tell Docker which port the app listens on
EXPOSE 3000

# Command that runs when the container starts
CMD ["node", "server.js"]

Cargo Manifest // 04

Commands you'll reach for first

  • docker pull <image>Download an image from Docker Hub to your machine.
  • docker build -t <name> .Build an image from a Dockerfile in the current folder.
  • docker run <image>Start a new container from an image.
  • docker psList containers that are currently running.
  • docker stop <id>Stop a running container by its ID or name.
  • docker imagesList every image stored locally.
